Firma PKCS#7 Apple Wallet
Problema detectado:
- El .pkpass nuevo tenia assets y manifest correctos.
- El certificado era el Pass Type ID correcto.
- Pero la firma PKCS#7 parecia generarse sin signed attributes, especialmente sin S/MIME signing-time.
- Apple Wallet requiere signing-time.

Correccion aplicada en PassGeneratorService:
- Usa SignedCms detached sobre manifest.json.
- Carga la coleccion completa del .p12.
- Usa como firmante el certificado con private key.
- Incluye el certificado Pass Type ID.
- Incluye certificado intermedio Apple WWDR G4.
- Agrega explicitamente:
  signer.SignedAttributes.Add(new Pkcs9SigningTime(DateTime.UtcNow))
- SignedCms agrega contentType y messageDigest.
- La salida DER se guarda como archivo signature.

WWDR G4
El .p12 local solo contiene el certificado Pass Type ID; no incluye el intermedio.
Se agrego el certificado publico Apple WWDR G4 al proyecto:
src\KBeauty.Loyalty.Infrastructure\Certificates\AppleWWDRCAG4.cer

Fuente oficial Apple:
https://www.apple.com/certificateauthority/

PassGeneratorService busca WWDR G4 en este orden:
1. Dentro del .p12.
2. Certificado empaquetado en AppContext.BaseDirectory\Certificates\AppleWWDRCAG4.cer.
3. Apple:WwdrCertificatePath si se configura.

Si no encuentra WWDR G4, lanza error claro indicando incluirlo en el .p12 o configurar Apple:WwdrCertificatePath.

ngrok http --url=goosepimply-quiana-unrudely.ngrok-free.dev https://localhost:55128

dotnet user-secrets set "Apple:WebServiceURL" "https://goosepimply-quiana-unrudely.ngrok-free.dev" --project src\KBeauty.Loyalty.API

Si se cambia WebServiceURL, reiniciar la API antes de generar el pass.
